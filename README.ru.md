# Seating Planner

Веб‑приложение для расстановки столов и рассадки гостей на мероприятиях (свадьбы, банкеты, конференции). Работает целиком в браузере: автосейв в `localStorage` плюс экспорт/импорт проекта в `.json`. Локализация EN/RU, светлая/тёмная тема, адаптив под десктоп, планшет и смартфон.

Стек: React + TypeScript + Vite, Konva (react‑konva) для холста, Zustand для состояния, Font Awesome Pro для иконок.

## Горячие клавиши

| Клавиши                             | Действие                                 |
|-------------------------------------|------------------------------------------|
| `Esc`                               | Снять выделение / закрыть окно           |
| `Ctrl/⌘ + A` / `Ctrl/⌘ + Shift + A` | Выделить все столы / снять выделение     |
| `Ctrl/⌘ + C` / `Ctrl/⌘ + V`         | Копировать / вставить (по курсору)       |
| `Ctrl/⌘ + D`                        | Дублировать                              |
| `Ctrl/⌘ + S`                        | Экспорт проекта в `.json`                |
| `Delete` / `Backspace`              | Удалить выбранное                        |
| Стрелки / `Shift` + стрелки         | Переместить мелким / крупным шагом       |
| `[` / `]` (с `Shift` — ±90°)        | Повернуть выбранное на ±15°              |
| `L`                                 | Заблокировать / разблокировать выбранное |
| `+` / `−` / `PageUp` / `PageDown`   | Зум; `0` — вписать в экран               |
| Пробел + перетаскивание             | Панорамирование холста                   |

Этот же список доступен в редакторе по кнопке справки (правый верхний угол).

## Токен Font Awesome Pro

Иконки используют Font Awesome Pro из приватного реестра, поэтому нужен токен **на этапе установки**.

- **Локально:** скопируйте `.env.example` в `.env` и задайте `FONTAWESOME_NPM_AUTH_TOKEN` (из аккаунта Font Awesome → *npm tokens*). Docker Compose читает `.env` автоматически.
- **CI (GitHub Actions):** добавьте секрет репозитория `FONTAWESOME_NPM_AUTH_TOKEN`.

## Запуск через Docker (без локального Node/npm)

```bash
docker compose up --build      # http://localhost:5173
```

Остановить — `docker compose down`. После изменения зависимостей пересоберите образ (`docker compose up --build`).

## Запуск через npm

```bash
npm install
npm run dev       # дев-сервер
npm run build     # сборка
npm run preview   # предпросмотр сборки
```
